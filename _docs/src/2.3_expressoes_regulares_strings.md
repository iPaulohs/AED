# Expressões regulares

As expressões regulares são uma forma de encontrar padrões no texto da string. Por exemplo, para verificar se um email é valido, podemos utilizar o padrão abaixo:

```csharp
using System.Text.RegularExpressions;

string texto = "mail@mail.com";
string pattern = @"\b[A-Za-z0-9_%+-]+(\.[A-Za-z0-9_%+-]+)*@[A-Za-z0-9-]+(\.[A-Za-z0-9-]+)*\.[A-Za-z]{2,}\b";

var match = Regex.IsMatch(texto, pattern);
Console.WriteLine(match); //True
```

### Entendo cada parte do padrão

O uso de `@` no início da string serve para evitar conflitos de `\` do compilador do C# com o motor do regex. Sem ele, seria necessário adicionar caracteres de escape para eviar o conflito.

`\b...\b` indica o início e o final do regex. tudo o que está entre os dois será considerado como parte do padrão a ser encontrado. 

`[...]` Essa estrutura entre colchetes é chamada tecnicamente de Character Class (Classe de Caracteres) ou Character Set. Ela indica ao motor do regex para validar um caractere por vez e verificar se o caractere está dentro da especificação. 

`A-Z` Indica que são permitidos caracteres maiúsculos de A até Z. 

`a-z` Indica que são permitidos caracteres minúsculo de a até z. 

`0-9` Indica que são permitidos dígitos de 0 a 9.` 

`_%+-` Permite a incidência dos caracteres especiais com um separados de dois blocos. 

`+` Significa que o padrão pode se repetir para cada caractere do texto.

`(\.[A-Za-z0-9_%+-]+)*` Os parênteses servem para repetir o bloco de verificação e o asterisco indica que quantas vezes forem necessárias. O `\.` é um ponto literal e serve para indicar que o ponto pode ser um separador dos blocos, mas apenas se utilizado uma vez:  
`mail.teste@maill.com ✅`  
`mail..teste@mail.com ❌`  

`@` O arroba indica que após os padrões anteriores, **deve** exist este símbolo naquele local.

`[A-Za-z0-9-]+(\.[A-Za-z0-9-]+)` Estre trecho valida a extensão do email. Conforme o padrão, ele deve ser composto apenas de letras e números. Caracteres especiais não são permitidos. Mais uma vez, o `()`e o `\.` indicam que o padrão pode se repetir e que apenas um `.` pode ser o separador.  
`mail.test@mail.com.br ✅`  
`mail.test@mail.123.br ❌`  
`mail.test@mail.com..br ❌` 

`*\.` Indica que, obrigatoriamente, deve existir um ponto como limitador. 


`[A-Za-z]{2,}` Repete a verificação de caracteres. As chaves indicam que o bloco deve ter, no mínimo, 2 caracteres.  
`mail.test@mail.com.br ✅`  
`mail.test@mail.com.b ❌`
